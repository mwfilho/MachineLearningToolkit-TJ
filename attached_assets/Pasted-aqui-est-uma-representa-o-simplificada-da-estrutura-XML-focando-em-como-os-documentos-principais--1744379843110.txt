Explicação para o Programador:
Nível Principal: A tag <processo> contém uma lista de tags <documento>. Cada <documento> representa um item principal adicionado ao processo (como uma petição, uma decisão, um ofício, etc.).
ID do Principal: Cada <documento> tem seu próprio idDocumento único.
Nível Vinculado (Anexos): Dentro de uma tag <documento>, pode haver uma lista de tags <documentoVinculado>. Estas representam os arquivos anexos àquele documento principal específico.
ID do Anexo: Cada <documentoVinculado> também tem seu próprio idDocumento único.
A Ligação Chave: A tag <documentoVinculado> possui o atributo idDocumentoVinculado. O valor deste atributo é sempre o idDocumento do <documento> pai ao qual ele está anexado. É assim que sabemos qual anexo pertence a qual documento principal.
Processamento Necessário: Para obter todos os arquivos, o código precisa:
Iterar pela lista de <documento> dentro de <processo>.
Para cada <documento>, pegar seu idDocumento e processá-lo (buscar conteúdo, etc.).
Dentro de cada <documento>, verificar se existe uma lista de <documentoVinculado>.
Se existir, iterar por essa lista.
Para cada <documentoVinculado>, pegar seu idDocumento (o ID dele mesmo, não o idDocumentoVinculado) e processá-lo (buscar conteúdo, etc.).
Essa estrutura aninhada e a referência idDocumentoVinculado são cruciais para reconstruir a organização correta dos documentos e seus anexos. O código atual precisa implementar essa lógica de iteração dupla (principal e depois vinculados).
