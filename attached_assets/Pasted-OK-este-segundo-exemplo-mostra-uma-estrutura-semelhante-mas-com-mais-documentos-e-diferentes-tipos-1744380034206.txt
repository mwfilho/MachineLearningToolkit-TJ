Pontos Chave para o Programador:

Lista Principal: A resposta da consulta consultarProcesso com incluirDocumentos=True retorna uma lista principal de documentos (processo.documento).

Documento Principal: Cada item nessa lista principal é um <documento>. Ele tem um idDocumento que o identifica unicamente.

Documentos Vinculados (Anexos): Dentro de cada <documento> principal, pode existir uma lista chamada documentoVinculado (o nome exato pode variar ligeiramente dependendo da biblioteca SOAP, mas a estrutura é essa). Cada item nessa lista interna é um <documentoVinculado>.

ID do Anexo: Cada <documentoVinculado> tem seu próprio idDocumento único.

Link Pai-Filho: O atributo idDocumentoVinculado dentro de <documentoVinculado> sempre aponta para o idDocumento do <documento> principal ao qual ele está anexado.

Necessidade de Iteração Dupla: Para processar todos os documentos (principais e anexos), o código precisa:

Iterar sobre a lista principal (processo.documento).

Para cada documento principal, pegar seu idDocumento e processá-lo (ex: chamar retorna_documento_processo para obter o conteúdo).

Verificar se esse documento principal contém a lista de documentoVinculado.

Se contiver, iterar sobre cada documentoVinculado.

Para cada documentoVinculado, pegar o idDocumento dele e processá-lo (ex: chamar retorna_documento_processo para obter seu conteúdo).

Este segundo exemplo confirma o padrão: um documento principal pode ter zero ou múltiplos documentos vinculados, e a ligação é feita pelo idDocumentoVinculado que referencia o idDocumento do pai.
